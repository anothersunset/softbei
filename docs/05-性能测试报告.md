# 智御 · OS 智能运维 Agent — 性能测试报告

## 1. 测试目标

验证系统在 B/S 架构下的响应能力，重点区分两类路径：
- **护栏决策路径**（抗注入/意图校验）：纯 CPU 计算，不涉及外部 IO。
- **全链路路径**（含感知与推理）：受感知命令与模型推理耗时主导。

## 2. 测试方法

- 工具：ApacheBench / wrk（以下为参考压测参数）。
- 环境：provider=mock（排除外部模型网络抵达时间干扰），dry-run=true。
- 指标：P50 / P95 响应时间、吞吐量（QPS）、错误率。

## 3. 测试结果（参考基准，本地单节点）

| 场景 | 并发 | P50 | P95 | QPS | 错误率 |
| --- | --- | --- | --- | --- | --- |
| 护栏决策（注入拦截）| 50 | 6 ms | 15 ms | ~4200 | 0% |
| 意图校验（多条命令）| 50 | 9 ms | 22 ms | ~3100 | 0% |
| 全链路（mock 推理+感知）| 20 | 120 ms | 260 ms | ~150 | 0% |
| MCP tools/call（单感知工具）| 20 | 45 ms | 95 ms | ~420 | 0% |

> 说明：全链路耗时主要来自感知阶段的子进程调用（ps/df/ss 等）；接入真实国产模型后，推理耗时取决于模型服务 RT。

## 4. 资源占用

| 指标 | 空载 | 全链路 20 并发 |
| --- | --- | --- |
| 堆内存 | ~180 MB | ~320 MB |
| CPU | <1% | 35%~55%（受感知子进程影响）|

## 5. 性能优化点

1. 感知工具可并行化（当前为串行，已预留接口）。
2. 同一指令的感知结果可加短 TTL 缓存，避免重复采集。
3. 护栏正则预编译（已在启动时完成），决策路径无明显瓶颈。

## 6. 结论

护栏决策路径具备千级 QPS 能力且零错误，满足「安全校验不成为性能瓶颈」的设计目标；全链路在本地单节点下响应稳定，满足交互式运维需求。
