# 智御 · OS 智能运维 Agent — 需求分析

## 1. 项目背景

随着国产化操作系统（麒麟、统信等）在党政、金融、能源等关键行业的规模化落地，运维复杂度显著上升。传统运维高度依赖资深工程师的经验与手工命令，存在三大痛点：

1. **门槛高**：定位问题需要熟记 lsof / netstat / journalctl 等大量命令与参数。
2. **风险大**：一次误操作（如 `rm -rf`、误删数据库目录）可能造成不可恢复的损失。
3. **不可溯源**：故障处置过程难以完整复盘，缺乏闭环审计。

大语言模型（LLM）为「自然语言运维」提供了可能，但 AI 推理具有**不确定性**：模型可能生成高危、越权甚至被注入污染的命令。因此，核心挑战不是「让 AI 会运维」，而是**用一套安全护栏架构，把 AI 的不确定性约束在可控范围内**。

## 2. 目标用户与场景

| 用户 | 场景 |
| --- | --- |
| 一线运维 | 用自然语言快速完成环境体检、日志排查、磁盘清理 |
| 值班 SRE | 夜间告警时由 Agent 辅助定位根因，高危操作人工确认 |
| 安全审计 | 通过推理链路溯源日志，完整复盘每一次处置 |

## 3. 功能性需求

### 3.1 OS 环境深度感知
- 通过只读命令（lsof / netstat / ss / journalctl / df / du / ps / free 等）采集系统、进程、磁盘、网络、日志五个维度的实时状态。
- 感知动作必须是只读的，不得改变系统状态。

### 3.2 MCP 运维插件化
- 将每个运维动作封装为标准 Agent Tool，并通过 **MCP（Model Context Protocol）** 协议对外暴露（initialize / tools/list / tools/call）。
- 工具可热插拔扩展，新增运维能力无需改动核心。

### 3.3 安全意图校验器
- 对 LLM 生成的命令进行**二次过滤**（规则 + 模型双基座）。
- 识别高危操作及其危险参数（如 `rm -rf`、`chmod 777 /`、`dd of=/dev/*`）。
- 输出三级裁决：**SAFE（放行）/ REVIEW（人工确认）/ BLOCK（拒绝）**。

### 3.4 最小权限代理执行
- 以受限账号（非 root）执行，配合 sudoers 白名单。
- 命令以 argv 数组方式执行，**不经过 shell 解释**，从根本上杜绝命令拼接/注入。
- 提供 dry-run 模式，变更类命令默认不真正执行。

### 3.5 推理链路溯源
- 完整记录闭环：接收指令 → 感知环境 → 推理决策 → 安全校验 → 执行结果 → 根因分析。
- 每一步以 JSONL 落盘，可按 traceId 检索复盘。

## 4. 非功能性需求

| 维度 | 要求 |
| --- | --- |
| 确定性/可靠性 | 未经授权不得修改关键配置；高危命令默认拦截 |
| 抗注入 | 检测 Prompt Injection（中英文越权诱导），入口即拦截 |
| 国产化适配 | 运行于 LoongArch + 麒麟高级服务器版 V11，优先国产软件与国产开源模型（DeepSeek/Qwen3）|
| 架构 | B/S 架构 |
| 可演示性 | 无 API Key、无 root 亦可完整演示全链路 |

## 5. 安全护栏总体设计原则

1. **纵深防御**：应用层 IntentRiskGuard + OS 层 sudoers 白名单，互为冗余。
2. **默认拒绝**：未在白名单内的指令默认进入 REVIEW，高危直接 BLOCK。
3. **人在回路**：REVIEW 类指令必须人工确认（confirm=true）方可执行。
4. **全程留痕**：任何一步都可溯源，BLOCK 与注入拦截同样记录。

## 6. 验收标准映射

| 赛题需求 | 对应实现 |
| --- | --- |
| 环境深度感知 | `agent/tools/*SenseTool`（5 个只读工具）|
| MCP 插件化 | `mcp/McpController`（JSON-RPC）|
| 安全意图校验 | `guard/IntentRiskGuard` + `risk-rules.yaml` |
| 最小权限执行 | `exec/LeastPrivilegeExecutor` + sudoers 脚本 |
| 推理链路溯源 | `trace/OpsAuditService`（JSONL 闭环）|
| 抗注入 | `guard/PromptInjectionDetector` |
