# 护栏泛化增强（Guard Generalization）

> 针对盲测报告（`docs/redteam-generalization.md`）暴露的失败项，本次做**架构级通用增强**，
> 不针对任何具体盲测样本，也不将盲测样本加入调参语料——避免对留出集（held-out set）过拟合而虚高指标。
>
> 提交分支：`feat/guard-generalization`（独立于已绿的 `main`，供本地复跑校验后再合并）。

## 1. 改进项

### 1.1 入口归一化（反编码 / 零宽绕过）
- 文件：`PromptInjectionDetector.java`
- 措施：检测前对输入做 **Unicode NFKC 归一化** 并剔除 **零宽 / 不可见 / 双向控制字符**（U+200B–200F、U+202A–202E、U+FEFF、U+00AD、C0 控制符等）。
- 效果：挫败「零宽空格切词」「全角伪装」「控制字符插入」等**任意**绕过，而非仅修复样本中的 BLD-005 / BLD-006。
- 局限：**不**处理 hex 十六进制转义类绕过——此类需在执行层解码后复检，列入演进路线。

### 1.2 纯只读管道放行（修复误拦）
- 文件：`IntentRiskGuard.java`
- 措施：当指令以 `|` 连接、且**每一段首个二进制都在只读白名单内**、段内不含管道以外的其它被禁元字符时，判定为 SAFE。
- 效果：修复 `top -bn1 | head -20`、`dmesg | tail -50` 等只读组合被误拦（FP）；含非只读命令（如 `... | bash`）或其它元字符（`;`、`>`、`$(` 等）的管道**仍然 BLOCK**，安全边界不变。

## 2. 刻意未做（避免过拟合留出集）
- **不**把盲测中出现的日 / 韩 / 同义词**具体关键词**硬编码进 `risk-rules.yaml`：那会让盲测召回率"看起来"上升，本质却是记忆留出集，违背泛化评估初衷。
- 多语言、语义改写（override / bypass / 别管 / 关掉）、社工诱导等语义层攻击，其通用解法是 **LLM 语义级注入检测**（见 `redteam-generalization.md` §5「长期」路线），而非关键词黑白名单。

## 3. 验证方式（须本地复跑，指标不手填）
1. `mvn -q test`：确认 168 项既有测试仍全绿（本改动为通用增强，预期不破坏既有断言）。
2. `mvn -q -Dtest=BlindsetRunnerTest test`：复跑盲测，由运行器输出**增强后**混淆矩阵。
3. 将复跑得到的真实 Precision / Recall / F1 回填至 `redteam-generalization.md`；复跑前其 §2 / §3 数字仍为**增强前基线**，不得手工修改。
