# 15 · 红蓝对抗注入语料 与 CI 工程化

> 本文说明三项工程化能力的方法与复现方式：① 红蓝对抗注入语料的确定性回放评测；② GitHub Actions 持续集成；③ JaCoCo 覆盖率报告。所有指标均由 `mvn test` 真实产出，本仓库不预置任何“漂亮数字”。

## 一、红蓝对抗注入语料（确定性回放）

- 语料文件：`backend/src/test/resources/redteam/injection-corpus.yaml`
- 回放用例：`backend/src/test/java/com/zhiqian/ops/eval/RedTeamCorpusTest.java`
- 评测对象：规则层 `PromptInjectionDetector`（入口抗注入）+ `IntentRiskGuard`（指令二次过滤），**不经过 LLM**，结果完全确定、可复现。

### 语料构成（共 48 条）

| 类别 | 子类 | 条数 | 预期 |
| --- | --- | --- | --- |
| 提示词注入 | 改写指令 / 越狱 / 泄露提示词 / 提权 / 反弹shell / 免密（中英） | 17 | 入口拦截 `blocked=true` |
| 正常对话 | 只读运维诉求（注入负样本） | 6 | 不误判 `blocked=false` |
| 危险命令 | `rm -rf`、`mkfs`、`dd`、关防火墙、改 `passwd`、元字符拼接等 | 10 | `BLOCK` |
| 变更命令 | `rm`/`mv`/`kill`/`systemctl restart`/`useradd`/未知二进制 | 7 | `REVIEW` |
| 只读命令 | `ls`/`df`/`free`/`ps`/`cat`/`tail`/`uptime` 等 | 8 | `SAFE` |

### 评测输出指标（运行 `mvn test` 时打印）

- 注入识别率（拦截率）= 命中 / 注入正样本
- 注入误拦率 = 正常对话被误判为注入 / 注入负样本
- 危险命令拦截率 = `BLOCK` 命中 / 危险命令
- 正常命令误拦率 = 只读命令被升级 / 只读命令

> 每条用例的预期由已固化的 `risk-rules.yaml` 规则唯一决定，测试以 `assertEquals(total, pass)` 断言**全部命中**。因此在 CI 中该语料的设计拦截率为 100%、设计误拦率为 0%，且**每次构建都会复现**。这是一条“会随规则回归的安全基线”，而非一次性截图。

### 如何扩充

向 YAML 追加 `type: injection`（含 `expectedBlocked`）或 `type: command`（含 `expectedLevel`）的用例即可，CI 下次运行自动纳入统计。**新增红队 payload 若被漏拦，CI 立即变红**——形成“红队加压 → 规则补强”的闭环。

## 二、持续集成（GitHub Actions）

- 工作流：`.github/workflows/ci.yml`
- 触发：`push` / `pull_request` 到 `main`
- 步骤：JDK 17（Temurin）→ `mvn -B -ntp clean test` → 上传 JaCoCo 覆盖率报告为构建产物
- 价值：每次提交自动回归全部单测（含安全护栏场景语料与红蓝对抗语料），保证“安全基线不回退”。

## 三、覆盖率（JaCoCo）

- 在 `backend/pom.xml` 引入 `jacoco-maven-plugin`，绑定 `prepare-agent` 与 `report`（test 阶段）。
- 运行 `mvn test` 后，HTML 报告位于 `backend/target/site/jacoco/index.html`，CI 中作为产物 `jacoco-report` 上传。
- 说明：真实覆盖率以本地 / CI 运行结果为准，本仓库**不预置覆盖率数字**。
