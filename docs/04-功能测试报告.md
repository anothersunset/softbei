## 4. 关键用例

| 类别 | 覆盖点 | 判定标准 |
|---|---|---|
| 安全护栏 | READONLY / EXECUTABLE / IRREVERSIBLE / BLOCK 四级裁决 | 危险命令与提示词注入稳定拦截，正常只读命令不误杀 |
| 人在回路 | OpsPipeline `traceId + confirm=true`；MCP `pendingMutationId + confirm=true` | 首轮直带确认不得执行，确认必须绑定已审阅的同一计划/参数 |
| 变更工具 | `service_restart` / `log_rotate` / `config_backup` | 护栏逐条裁决、dry-run 可复现、执行前备份、回滚账本、trace 审计 |
| 路径安全 | `log_rotate` 允许范围与真实路径校验 | live 模式拒绝不存在文件、非普通文件、符号链接和越权路径 |
| 溯源 | JSONL 轮转与历史回扫 | LRU 未命中、重启后、归档后仍可按 traceId 查询 |
| 感知工具 | system/process/disk/network/log/db/metrics/health | 只读标注、参数 schema、无凭据降级与 SSRF 限制均生效 |

## 5. 验收脚本

龙芯、麒麟 V11 或 x86 Ubuntu 服务器复测请执行：

```bash
bash scripts/loongarch-full-acceptance.sh
```

脚本默认拉取 `main`，报告写入 `acceptance-runs/` 或脚本输出的 `REPORT_DIR`。真实 LLM 复测需额外设置 `OPS_LLM_API_KEY`，不得把 `.env` 或 key 放进提交压缩包。
