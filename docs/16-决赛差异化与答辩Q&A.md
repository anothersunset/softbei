# 16 · 决赛差异化与模拟答辩

> 面向“软件杯”决赛（截止 2026-07-20）。本文聚焦：我们与“普通 LLM 运维助手”的本质差异、评分维度对照、以及评委高频问答的应对话术。文中只引用仓库内**已实现且可复现**的能力，不含夸大数据。

## 一、一句话定位

OpsGuard 智御 = 以“安全护栏”为第一性的 OS 智能运维 Agent：**能感知、敢执行、可回滚、零失控**。区别于“把命令丢给大模型直接跑”的方案，我们在大模型前后各加了一道确定性安全闸门。

## 二、与普通方案的差异化对照

| 维度 | 普通 LLM 运维助手 | OpsGuard 智御 |
| --- | --- | --- |
| 提示词注入 | 依赖模型自觉 | 入口确定性正则拦截（`PromptInjectionDetector`），中英越权 / 越狱模式库 |
| 危险命令 | 模型判断，易被绕过 | 规则二次过滤（`IntentRiskGuard`）：元字符 → 红线正则 → 关键路径 → 变更白名单，`SAFE`/`REVIEW`/`BLOCK` 三态 |
| 执行边界 | 直接执行 | 受限执行 + dry-run + 最小权限，变更类强制人工确认 |
| 出事之后 | 难追溯 / 难回滚 | 动作账本 + 一键回滚建议（`RollbackAdvisor`），全链路 trace |
| 可解释性 | 黑箱 | 安全评分三维（静态 30 / 动态 35 / 受限执行 35）+ 反事实回放 |
| 根因定位 | 单点日志 | 跨源 RCA：指标↔日志窗口关联 + L1–L3，LLM 可选增强、可 mock 回退 |
| 协议生态 | 自定义接口 | MCP 双通道（HTTP + stdio），JSON-RPC 2.0 合规 |
| 工程可信度 | demo 截图 | CI 持续回归 + 安全 / 红蓝语料随构建复现 + JaCoCo 覆盖率 |

## 三、对应官方评分维度的“证据锚点”

（仅列已实现 / 可复现项；具体数值以现场 `mvn test` 与部署验收为准）

- **功能完整性**：8 段管线（RECEIVE → INJECTION_GUARD → SENSE → RETRIEVE → REASON → GUARD → EXECUTE → ANALYZE）
- **安全性**：注入语料 + 红蓝对抗语料确定性回放；三态裁决；关防火墙 / 改 passwd / 元字符拼接等红线
- **可靠性**：执行轮次熔断（`OPS_EXEC_MAX_STEPS`）、dry-run、回滚
- **创新性**：安全评分 + 反事实回放、跨源 RCA、MCP stdio 双通道
- **工程化**：CI、覆盖率、文档体系（`docs/01`–`16`）

## 四、模拟答辩 Q&A

**Q1：你们和直接调用大模型跑命令的区别？**
A：我们把大模型当“建议者”而非“执行者”。模型前有抗注入闸门，模型后有意图风险校验与受限执行；任何变更默认 `REVIEW`，红线命令永不执行。**模型不可信时系统依然安全。**

**Q2：规则会不会太死、误杀正常操作？**
A：规则分三态——只读白名单直接放行（`SAFE`），变更类走人工确认（`REVIEW`），仅红线才 `BLOCK`。我们用注入负样本与只读命令样本在 CI 中持续度量误拦率，目标 0 误杀；规则与 `risk-rules.yaml` 解耦可热调。

**Q3：拦截率怎么证明，不是 PPT 数字？**
A：`RedTeamCorpusTest` 把 48 条红蓝语料随 `mvn test` 全量回放并断言，CI 每次构建复现；漏拦即构建变红。这是“会回归的安全基线”，可现场跑给评委看。

**Q4：模型被注入诱导“忽略规则”怎么办？**
A：注入识别在入口，独立于模型；命中即 `INJECTION_BLOCKED`，根本进不到执行层。中英文越权 / 越狱 / 泄露提示词 / 反弹 shell / base64 解码 / sudoers 免密等模式均在库内。

**Q5：执行出问题如何兜底？**
A：动作账本记录每步，`RollbackAdvisor` 给出回滚建议并支持一键回滚；执行有轮次熔断防止失控循环；变更默认 dry-run。

**Q6：根因分析是不是套壳大模型？**
A：RCA 先做确定性的跨源关联（指标异常窗口↔日志），给出 L1–L3 分层判断；LLM 仅作可选增强，且可 mock 回退，断网 / 无 key 也能给出根因骨架。

**Q7：为什么用 MCP？**
A：MCP 是模型-工具的标准协议。我们实现 HTTP + stdio 双通道并通过 JSON-RPC 2.0 合规校验，便于被任意 MCP 客户端编排，生态可扩展。

**Q8：安全评分的权重依据？**
A：对齐“多层级安全”理念：静态风险评估 30% + 动态意图审计 35% + 受限执行 35%，纯确定性计算，不影响管线裁决，仅用于可解释呈现，并配反事实回放（“如果没有护栏会怎样”）。
