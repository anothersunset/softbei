# 智御 · OS 智能运维 Agent — 安装部署文档

## 1. 运行环境

| 项 | 要求 |
| --- | --- |
| 架构 | LoongArch (loong64) |
| 操作系统 | 麒麟高级服务器版 V11 |
| JDK | 17（推荐国产 毕昇 BiSheng JDK / Loongson JDK）|
| 构建 | Maven 3.8+ |
| 模型（可选）| DeepSeek / Qwen3（国产开源模型）|

## 2. 目录结构

```
softbei/
├── backend/                      # Spring Boot 后端（含静态控制台）
│   ├── pom.xml
│   └── src/main/
│       ├── java/com/zhiqian/ops/ # 源代码
│       └── resources/
│           ├── application.yml    # 主配置
│           ├── risk-rules.yaml    # 安全规则库（配置化，重启生效）
│           └── static/index.html  # B/S 前端控制台
├── deploy/                        # 部署文件
│   ├── Dockerfile.loong64
│   ├── docker-compose.loong64.yml
│   └── scripts/                   # 最小权限账号脚本
└── docs/                          # 提交文档
```

## 3. 方式一：本地构建运行（最快验证）

```bash
cd backend
mvn -B clean package -DskipTests
java -jar target/ops-agent-1.0.0.jar
```
打开浏览器访问 `http://127.0.0.1:8080/`。

> 默认 `provider=mock` 且 `dry-run=true`，**无需 API Key、无需 root** 即可完整演示七阶段闭环。

### 信任边界

默认服务仅绑定 `127.0.0.1`，用于本机演示。内网或公网部署时必须显式设置：

```bash
export OPS_BIND_ADDRESS=0.0.0.0
export OPS_API_TOKEN=<强随机令牌>
```

启用令牌后，REST `/api/**` 与 HTTP MCP `/mcp/**` 均需携带 `X-Ops-Token` 或 `Authorization: Bearer`。如果绑定到非回环地址但未配置令牌，服务会拒绝启动。

## 4. 方式二：容器化部署

```bash
# 项目根目录执行
docker compose -f deploy/docker-compose.loong64.yml up -d --build
```
镜像基于国产 openEuler/麒麟系基础镜像 + JDK17，容器内以受限账号 `opsagent` 运行服务进程。

## 5. 接入国产大模型（可选）

编辑 `backend/src/main/resources/application.yml`：
```yaml
ops:
  llm:
    provider: deepseek
    base-url: https://api.deepseek.com
    model: deepseek-chat
    api-key: <你的Key>
```
或通过环境变量 `OPS_LLM_PROVIDER` / `OPS_LLM_API-KEY` 覆盖。

## 6. 启用真实执行与最小权限（生产）

```bash
sudo OPS_USER=opsagent bash deploy/scripts/setup-ops-user.sh
```
该脚本将：
1. 创建受限账号 `opsagent`（非 root）。
2. 安装 `sudoers` 白名单到 `/etc/sudoers.d/ops-agent`，并 `visudo -c` 校验。

随后修改配置：
```yaml
ops:
  exec:
    dry-run: false      # 谨慎：关闭后变更类命令将真正执行
    use-sudo: true
    run-as-user: opsagent
```

> **安全双重防护**：应用层 `IntentRiskGuard` 负责语义级拦截；OS 层 `sudoers` 白名单负责权限级兜底。即使应用层被绕过，OS 层仍限制可执行范围。

## 7. 健康检查

```bash
curl http://127.0.0.1:8080/actuator/health
curl http://127.0.0.1:8080/api/ops/tools
```

## 8. 常见问题

| 现象 | 处理 |
| --- | --- |
| 感知命令报「command not found」| 麒麟 V11 默认未装 `ss/lsof`，执行 `yum install -y iproute lsof` |
| sudoers 路径不符 | 用 `which journalctl ss` 确认实际路径后修改 `sudoers-ops-agent` |
| 端口冲突 | 修改 `application.yml` 的 `server.port` |
