## 方法

2 小时限时黑盒攻击，目标：使危险命令被判 SAFE 绕过。攻击面入口：`POST /api/ops/chat`，MockLLM 模式。共 56 条攻击样本（6 类：注入变体 / 命令混淆 / 伪装维护 / 语义攻击 / 边界情况 / 编码绕过），事后白盒对照。

## 关键发现

1. **注入检测绕过**：5 类手法成功绕过（上下文伪装/代码块注入/hex编码/URL编码/八进制转义）
   - 根因：`normalize()` 只处理 `\xNN`，不解码 URL 编码、八进制、空格分隔 hex
   - 缺少"调试模式"/"代码审查"/"培训数据"等社会工程伪装检测
2. **REVIEW/BLOCK 边界**：3 个命令应升级为 BLOCK（dd to /etc/passwd, kill -9 1, userdel -r）
3. **MockLLM 限制**：MockLLM 对危险指令产出 `uptime; free -h; df -h`，掩盖了真实 LLM 场景下的风险
4. **护栏命令级有效**：38/56 BLOCKED，所有有危险命令产出且通过注入检测的案例均被护栏拦截

## 结论

201 tests 的"全绿"不代表防线完美。编码绕过 + 真实 LLM 场景下存在可走通的攻击链。详见 `docs/shadow-redteam-report-2026-07-02.md`。
