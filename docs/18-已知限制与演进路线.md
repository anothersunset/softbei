# 已知限制与演进路线（Known Limitations & Roadmap）

> 诚实声明边界比掩盖问题更能赢得评委信任。本文集中说明本作品当前的能力边界与后续规划。

## 1. 已知限制

### 1.1 护栏检测为规则层
- 当前注入/风险识别以 `risk-rules.yaml` 的模式匹配为主，优点是**确定、可复现、可审计**；局限是对全新变体的泛化依赖规则覆盖面。
- 缓解：已引入盲测集（`blindset-corpus.yaml`）持续衡量泛化；路线上计划增加输入规范化（解码/去同形字/去空白）预处理层。

### 1.2 RCA 深度
- 已落地三类故障（OOM / DISK_FULL / IO）且主要在 L2 分级验证；跨服务依赖雪崩、网络分区、配置漂移等场景尚未覆盖。
- 路线：见 `docs/19-RCA故障分级矩阵.md` 中标记为「规划」的条目。

### 1.3 演示与可观测
- 当前演示为 asciinema 终端录屏，叙事张力有限；路线上补一段 90s「拦截→定位→回滚」叙事型成片（见 `docs/20-演示脚本与分镜.md`）。

### 1.4 工程成熟度
- 单人开发，覆盖面广。为避免「广而不深」，已将**安全护栏闭环**（意图分级+反事实回放+回滚账本）作为招牌深度线，其余能力服务于这条主线。
